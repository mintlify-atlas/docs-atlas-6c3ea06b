## Overview

The DossPanels REST API uses **Laravel Passport** OAuth2 Bearer tokens for authentication. All protected endpoints require you to include the token in the `Authorization` header of every request.

```
Authorization: Bearer {your_access_token}
```

**Base URL:** `https://yourdomain.com/api`

## Login

Obtain an access token by submitting user credentials to the login endpoint.

```
POST /api/login
```

### Request parameters

<ParamField body="email" type="string" required>
  The user's registered email address.
</ParamField>

<ParamField body="password" type="string" required>
  The user's password.
</ParamField>

<ParamField body="phone" type="string">
  Phone number (alternative to email for phone-based login).
</ParamField>

### Response

<ResponseField name="status" type="integer">
  HTTP status code. `200` on success.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable response message.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="data properties">
    <ResponseField name="access_token" type="string">
      Bearer token to use in subsequent API requests.
    </ResponseField>
    <ResponseField name="token_type" type="string">
      Always `"Bearer"`.
    </ResponseField>
    <ResponseField name="user" type="object">
      Basic user profile information.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl -X POST https://yourdomain.com/api/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "yourpassword"
  }'
```

```javascript JavaScript
const response = await fetch('https://yourdomain.com/api/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'yourpassword'
  })
});
const data = await response.json();
const token = data.data.access_token;
```
</CodeGroup>

### Error responses

| Status | Meaning |
|--------|---------|
| `401` | Invalid credentials |
| `403` | Account suspended or inactive |
| `422` | Validation error (missing fields) |

## Using the token

Include the access token as a Bearer token in the `Authorization` header for all authenticated requests:

```bash
curl https://yourdomain.com/api/get-user-profile \
  -H "Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJSUzI1..."
```

## Logout

Revoke the current access token.

```
POST /api/logout
```

Requires `Authorization: Bearer {token}` header. Returns `200` on success.

## Additional authentication endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/get-preference-settings` | Retrieve platform preference settings (public) |
| `GET` | `/api/get-paymoney-settings` | Retrieve payment platform settings (public) |
| `GET` | `/api/check-user-status` | Check if a user account is active |
| `GET` | `/api/check-login-via` | Determine whether login should use email or phone |

<Note>
All endpoints except login, registration, and a few public settings endpoints require a valid Bearer token. Requests without a valid token return `401 Unauthorized`.
</Note>
