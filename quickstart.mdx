---
title: "Quickstart"
description: "Get DossPanels running and make your first transaction in under 15 minutes."
---

This guide walks you through cloning the repository, configuring the environment, and completing your first transfer between two users.

<Steps>
  <Step title="Clone the repository">
    Clone the DossPanels source to your local machine or server:

    ```bash
    git clone <your-repository-url> dosspanels
    cd dosspanels
    ```
  </Step>

  <Step title="Install PHP dependencies">
    Install all Composer packages, including Laravel Passport, Sanctum, Firebase, and the module system:

    ```bash
    composer install
    ```

    <Note>
      DossPanels requires PHP 8.1 or higher. Run `php -v` to confirm your version before proceeding.
    </Note>
  </Step>

  <Step title="Configure the environment">
    Copy the example environment file and generate an application key:

    ```bash
    cp .env.example .env
    php artisan key:generate
    ```

    Open `.env` and update the following values at minimum:

    ```env
    APP_NAME=DossPanels
    APP_URL=http://your-domain.com
    APP_INSTALL=true

    DB_DATABASE=dosspanels
    DB_USERNAME=your_db_user
    DB_PASSWORD=your_db_password
    ```

    <Info>
      Set `APP_INSTALL=true` so the application bootstraps transaction type and payment method constants on startup. Without this, the platform will not function correctly.
    </Info>
  </Step>

  <Step title="Run migrations and seed the database">
    Create all tables and populate them with the required seed data, including the default admin account, transaction types, currencies, and roles:

    ```bash
    php artisan migrate --seed
    ```

    This inserts the default admin account with the credentials:
    - **Email:** `admin@techvill.net`
    - **Password:** `123456`

    <Warning>
      Change the default admin password immediately after your first login.
    </Warning>
  </Step>

  <Step title="Install Laravel Passport">
    Passport provides the OAuth2 server that powers API authentication:

    ```bash
    php artisan passport:install
    ```

    This generates the encryption keys and creates the default OAuth clients in the database.
  </Step>

  <Step title="Create the storage symlink">
    Link the public storage directory so uploaded files (avatars, logos, bank documents) are accessible:

    ```bash
    php artisan storage:link
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    php artisan serve
    ```

    The application is now available at `http://localhost:8000`.
  </Step>

  <Step title="Log in to the admin panel">
    Navigate to `http://localhost:8000/admin` and sign in with:
    - **Email:** `admin@techvill.net`
    - **Password:** `123456`

    From the admin dashboard you can manage currencies, users, fees, and transactions.
  </Step>

  <Step title="Create your first currency">
    Before users can hold wallets, at least one active currency must exist.

    1. In the admin panel, go to **Currency Management**.
    2. Click **Add Currency**.
    3. Set the type to **Fiat**, enter a name (e.g. `US Dollar`), symbol (`$`), and code (`USD`).
    4. Set the rate to `1` for the base currency.
    5. Set status to **Active** and mark it as the default currency.
    6. Save the currency.
  </Step>

  <Step title="Register the first user">
    Navigate to the user registration page at `http://localhost:8000/register` and create a new account. Alternatively, create a user directly from the admin panel under **User Management > Add User**.

    When a user registers, DossPanels automatically creates a wallet for the default currency with a starting balance of `0`.
  </Step>

  <Step title="Make your first transfer">
    1. Log in as the user you just created.
    2. First, add funds by navigating to **Deposit** and submitting a deposit request. An admin must approve it.
    3. Log back into the admin panel and approve the pending deposit under **Deposits**.
    4. Return to the user dashboard and navigate to **Send Money**.
    5. Enter a recipient email or phone number, an amount, and submit.

    The transaction appears in both users' transaction histories with status `Success`.
  </Step>
</Steps>
