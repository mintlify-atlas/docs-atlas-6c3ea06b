## Server requirements

Before installing DossPanels, confirm your server meets the following requirements.

<AccordionGroup>
  <Accordion title="PHP and extensions">
    - **PHP 8.1** or higher
    - Extensions: `bcmath`, `ctype`, `fileinfo`, `json`, `mbstring`, `openssl`, `pdo`, `pdo_mysql`, `tokenizer`, `xml`, `gd`, `zip`, `curl`, `intl`
    - The `intl` extension is required by `giggsey/libphonenumber-for-php` for phone number parsing.
    - The `gd` extension is required by `intervention/image` for image processing.
  </Accordion>

  <Accordion title="Database and cache">
    - **MySQL 5.7+** or **MySQL 8.0+** (recommended)
    - **Redis** — used for queue workers and caching in production
    - Set `QUEUE_CONNECTION=redis` in production to process jobs asynchronously
  </Accordion>

  <Accordion title="Node.js and build tools">
    - **Node.js 18+** and **npm** — required to build frontend assets with Vite
    - Run `npm install && npm run build` after deployment
  </Accordion>

  <Accordion title="Composer">
    - **Composer 2.x** — PHP dependency manager
  </Accordion>
</AccordionGroup>

## Install from source

<Steps>
  <Step title="Clone and install dependencies">
    ```bash
    git clone <your-repository-url> dosspanels
    cd dosspanels
    composer install --optimize-autoloader --no-dev
    npm install && npm run build
    ```
  </Step>

  <Step title="Set up the environment file">
    ```bash
    cp .env.example .env
    php artisan key:generate
    ```
  </Step>

  <Step title="Run migrations and seed">
    ```bash
    php artisan migrate --seed
    php artisan passport:install
    ```
  </Step>

  <Step title="Create the storage symlink">
    ```bash
    php artisan storage:link
    ```
  </Step>

  <Step title="Set directory permissions">
    ```bash
    chmod -R 775 storage bootstrap/cache
    chown -R www-data:www-data storage bootstrap/cache
    ```
  </Step>

  <Step title="Optimize for production">
    ```bash
    php artisan config:cache
    php artisan route:cache
    php artisan view:cache
    ```
  </Step>
</Steps>

## Environment configuration

All configuration is done through the `.env` file. The table below documents every key variable from `.env.example`.

### Application

```env
APP_NAME=DossPanels
APP_ENV=production
APP_KEY=                        # Generated by php artisan key:generate
APP_DEBUG=false
APP_URL=https://your-domain.com
APP_INSTALL=true                # Must be true for the platform to bootstrap correctly
```

<Warning>
  Set `APP_DEBUG=false` in production. Leaving it `true` exposes stack traces and environment variables to end users.
</Warning>

### Database

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=dosspanels
DB_USERNAME=dosspanels_user
DB_PASSWORD=strong_password_here
```

### Cache, sessions, and queues

```env
BROADCAST_DRIVER=pusher         # Use 'log' for local dev without Pusher
CACHE_DRIVER=redis              # Use 'file' for single-server setups
FILESYSTEM_DISK=local
QUEUE_CONNECTION=redis          # Use 'sync' for local dev (no background workers needed)
SESSION_DRIVER=redis
SESSION_LIFETIME=120
```

### Redis

```env
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379
```

### Mail

```env
MAIL_MAILER=smtp
MAIL_HOST=smtp.your-provider.com
MAIL_PORT=587
MAIL_USERNAME=your@email.com
MAIL_PASSWORD=your_mail_password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS="noreply@your-domain.com"
MAIL_FROM_NAME="DossPanels"
```

### Pusher (real-time notifications)

DossPanels uses Pusher for real-time browser notifications. Create an account at [pusher.com](https://pusher.com) and fill in these values:

```env
PUSHER_APP_ID=your_app_id
PUSHER_APP_KEY=your_app_key
PUSHER_APP_SECRET=your_app_secret
PUSHER_HOST=
PUSHER_PORT=443
PUSHER_SCHEME=https
PUSHER_APP_CLUSTER=mt1

VITE_PUSHER_APP_KEY="${PUSHER_APP_KEY}"
VITE_PUSHER_HOST="${PUSHER_HOST}"
VITE_PUSHER_PORT="${PUSHER_PORT}"
VITE_PUSHER_SCHEME="${PUSHER_SCHEME}"
VITE_PUSHER_APP_CLUSTER="${PUSHER_APP_CLUSTER}"
```

### AWS S3 (optional file storage)

To store user uploads in S3 rather than local disk, set `FILESYSTEM_DISK=s3` and provide:

```env
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_DEFAULT_REGION=us-east-1
AWS_BUCKET=your-bucket-name
AWS_USE_PATH_STYLE_ENDPOINT=false
```

## Queue workers

DossPanels dispatches background jobs for OTP codes, notifications, and transaction processing. In production, run a queue worker as a long-running process.

```bash
php artisan queue:work --sleep=3 --tries=3
```

Use a process manager like **Supervisor** to keep the worker running:

```ini
[program:dosspanels-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/dosspanels/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
user=www-data
numprocs=4
redirect_stderr=true
stdout_logfile=/var/www/dosspanels/storage/logs/worker.log
stopwaitsecs=3600
```

## Web server configuration

<Tabs>
  <Tab title="nginx">
    ```nginx
    server {
        listen 80;
        server_name your-domain.com;
        root /var/www/dosspanels/public;

        add_header X-Frame-Options "SAMEORIGIN";
        add_header X-Content-Type-Options "nosniff";

        index index.php;

        charset utf-8;

        location / {
            try_files $uri $uri/ /index.php?$query_string;
        }

        location = /favicon.ico { access_log off; log_not_found off; }
        location = /robots.txt  { access_log off; log_not_found off; }

        error_page 404 /index.php;

        location ~ \.php$ {
            fastcgi_pass unix:/var/run/php/php8.1-fpm.sock;
            fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
            include fastcgi_params;
        }

        location ~ /\.(?!well-known).* {
            deny all;
        }
    }
    ```
  </Tab>

  <Tab title="Apache">
    ```apacheconf
    <VirtualHost *:80>
        ServerName your-domain.com
        DocumentRoot /var/www/dosspanels/public

        <Directory /var/www/dosspanels/public>
            AllowOverride All
            Require all granted
        </Directory>

        ErrorLog ${APACHE_LOG_DIR}/dosspanels-error.log
        CustomLog ${APACHE_LOG_DIR}/dosspanels-access.log combined
    </VirtualHost>
    ```

    Ensure `mod_rewrite` is enabled:

    ```bash
    a2enmod rewrite
    systemctl restart apache2
    ```
  </Tab>
</Tabs>

<Note>
  DossPanels requires the document root to point to the `public/` subdirectory, not the project root.
</Note>

## Firebase setup (push notifications)

DossPanels uses `kreait/firebase-php` for mobile push notifications. To enable this:

1. Create a project in the [Firebase Console](https://console.firebase.google.com).
2. Download the service account JSON file from **Project Settings > Service Accounts**.
3. Place the file in a secure location outside the web root.
4. In the admin panel, navigate to **Settings > Firebase** and upload or reference the service account credentials.

## Post-installation checklist

- [ ] `APP_DEBUG=false` in `.env`
- [ ] Default admin password changed
- [ ] Queue workers running via Supervisor
- [ ] Storage symlink created (`php artisan storage:link`)
- [ ] At least one active currency configured
- [ ] SSL certificate installed (Let's Encrypt or equivalent)
- [ ] `php artisan config:cache` run after finalising `.env`
